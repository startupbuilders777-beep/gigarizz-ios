# GigaRizz V2 Product Plan

## Mission

GigaRizz V2 should become the AI profile studio that turns a user's existing photos into a dating profile that gets more matches.

The product should stop feeling like a collection of AI tools and start feeling like one guided outcome:

> Upload your current photos. Get a better dating profile today.

The app should own the full profile upgrade loop:

1. Audit what the user has.
2. Explain what is missing.
3. Generate or improve the missing photos.
4. Assemble a platform-specific profile kit.
5. Help the user message once matches arrive.

## Positioning

### App Store Category Position

Primary positioning:

> AI Dating Photos & Profile Coach

Subtitle options:

- Get better photos for Hinge, Tinder & Bumble
- Fix your dating profile with AI
- Photos, bios, prompts & openers

Short pitch:

> GigaRizz analyzes your dating photos, finds what is hurting your profile, and creates better photos, bios, prompts, and openers for Hinge, Tinder, and Bumble.

### Positioning Principles

- Lead with the real user pain: bad dating photos, weak profiles, low matches.
- Use "rizz" as brand flavor, not as the main App Store promise.
- Avoid exposing model/provider names in the primary UX.
- Avoid unsubstantiated claims like "3x more matches" unless backed by real product data.
- Promise improvement and clarity, not a fake identity.

### Trust Position

This category has an authenticity problem. Users worry that AI dating photos will look fake or misrepresent them.

GigaRizz should explicitly own this:

> Better photos, not a fake person.

Trust features should include:

- "Keep me looking like me" toggle, default on.
- Natural dating photo quality check.
- Identity drift warnings.
- "Too AI-looking" detector.
- No fake luxury, travel, or lifestyle claims by default.
- Clear upload privacy explainer.
- Delete all photos/data control.

## Competitive Reference

GigaRizz should learn from competitors, but not copy their exact surface area.

- Photofeeler: strong reference for photo feedback and trait scoring. It is good at helping users stop guessing, but it does not generate the missing assets.
- YourMove AI: strong reference for profile writing and texting assistance. It is outcome-oriented around reducing dating app exhaustion.
- CupidAI: strong reference for screenshot-based opener/reply flows and full-stack dating assistance.
- Tinder Photo Selector: strong reference for reducing photo-selection friction and positioning AI as user-controlled assistance.

The opening is to combine these into a single profile upgrade workflow:

> Audit like Photofeeler, generate the missing photo set, write like YourMove/CupidAI, and package everything for Hinge/Tinder/Bumble.

## V2 Product Spine

The main V2 flow should be:

> Profile Upgrade Flow

Fresh users should not start in a tools hub or a long onboarding story. They should immediately enter the problem they came to solve.

### First Run Flow

1. Ask: "What do you want to improve?"
2. Let the user choose target apps: Hinge, Tinder, Bumble, Raya, Other.
3. Ask the user to upload 4-8 current dating photos.
4. Audit the current photo set.
5. Show a Profile Diagnosis.
6. Offer to generate the full Upgrade Kit.

Suggested first question:

> What do you want to improve?

Options:

- More matches
- Better first photo
- Better Hinge profile
- Better Tinder/Bumble photos
- Better openers/replies

### Magic Moment

The magic moment is not seeing a landing screen. It is seeing:

- The user's best current photo.
- The user's weakest current photo.
- The top 3 reasons the profile is underperforming.
- A concrete plan to fix it.

Target:

- Time from install to diagnosis: under 2 minutes.
- Time from install to saved profile kit: under 5 minutes in mock/demo mode, under 10 minutes with real generation.

## The Upgrade Kit

The Upgrade Kit should be the hero artifact of V2.

A completed kit should include:

- Best first photo.
- Recommended 4-6 photo order.
- Generated/improved alternatives.
- Hinge prompt answers.
- Tinder/Bumble bio.
- Opening message examples.
- Export checklist.

This is stronger than a gallery of generated images. The user wants a finished dating profile, not a pile of assets.

## Tool Consolidation

Current standalone tools should become contextual actions.

### Move Into Photo Actions

- Face Enhance -> Improve this photo.
- Outfit Studio -> Upgrade outfit.
- Hairstyle -> Try new look.
- Background Replacer -> Improve setting.
- Pose Studio -> Generate this slot.
- Color Grade -> Improve lighting.

### Move Into Diagnosis

- Photo Ranking -> Recommended photo order.
- Photo Audit -> Profile Diagnosis.
- Expression Coach -> Face/expression feedback.

### Deprioritize For V2

- Age Studio.
- Pose Library as standalone surface.
- Advanced model picker.
- Batch model comparison.
- Match tracker as a primary tab.
- 30-slide onboarding.

These can return later if they support the core Profile Upgrade Flow.

## Core V2 Screens

### Profile Diagnosis

Purpose:

Tell the user what is working, what is hurting them, and what to do next.

Content:

- Overall profile score.
- Best current photo.
- Weakest current photo.
- Missing photo archetypes.
- Platform-specific recommendations.
- Top fixes.

Tone:

- Tactical.
- Specific.
- Not shamey.
- Not vague.

Example:

> Your first photo is clear, but your profile lacks social proof and lifestyle variety. Hinge profiles usually perform better with a strong face shot, a candid activity photo, and one personality cue.

### Missing Photo Slots

Instead of asking the user to choose a model or style, show profile slots:

- First Photo.
- Casual Candid.
- Dressed Up.
- Hobby/Activity.
- Travel/Lifestyle.
- Social Proof.

Each slot should explain why it matters and provide a Generate or Improve action.

### Improve This Photo

When viewing a photo, actions should be:

- Make more natural.
- Improve lighting.
- Change background.
- Upgrade outfit.
- Try new hairstyle.
- Crop for Hinge.
- Crop for Tinder.
- Rank against others.

### Export Profile Kit

The final output should be concrete:

- Photo order for Hinge.
- Photo order for Tinder.
- Photo order for Bumble.
- Bio.
- Prompt answers.
- First messages.
- Save all.
- Copy all.
- Share/export.

## Conversion Strategy

### App Store Screenshots

The screenshot narrative should be pain -> diagnosis -> transformation -> profile kit -> messaging help.

1. Fix Your Dating Profile
   - Show audit score and weak photo callout.

2. Find What's Costing You Matches
   - Show photo issues like bad lighting, weak first photo, no variety.

3. Generate Better Dating Photos
   - Show believable before/after.

4. Build Your Hinge/Tinder Kit
   - Show ordered photos, bio, prompts.

5. Get Better Openers
   - Show screenshot-to-reply flow.

### App Store Keywords

Primary keyword targets:

- dating photos
- AI dating photos
- dating profile
- Hinge profile
- Tinder photos
- Bumble profile
- dating coach
- AI bio writer
- rizz app
- opener generator

### Paywall Timing

Paywall should appear after value is visible.

Bad:

> Unlock full power before the user understands the diagnosis.

Better:

> We found 3 issues hurting your profile. Generate your full upgrade kit.

The best paywall trigger is after the free diagnosis, when the user sees specific problems and the app offers a specific fix.

## Pricing Strategy

Plans should be simple and tied to user-visible value.

### Free

- 1 profile audit.
- 1-2 generated previews.
- Limited bio/openers.

### Plus

- Full profile kit.
- Monthly or daily photo generation quota.
- Bio/prompts/openers.
- HD exports.

### Pro

- Higher/unlimited generation cap.
- Advanced photo edits.
- Priority generation.
- Weekly profile refresh.
- Screenshot reply coach.

Value metric should be profile kits and photo generations, not access to obscure model names.

## Technical Plan

### Phase 1: Product Refactor

Goal:

Make the app shape match the V2 mission.

Tasks:

- Replace tab structure with Upgrade, Photos, Coach, Profile.
- Collapse ToolsHub into contextual photo actions.
- Replace 30-step onboarding with first-run profile audit.
- Make Profile Audit the core entry point.
- Make feature flags actually control onboarding and paywall behavior.
- Remove or hide advanced model controls from primary flows.

### Phase 2: Unified Profile Kit Model

Create a shared model that every major feature reads from or writes to.

Suggested shape:

```swift
struct ProfileKit: Codable, Identifiable {
    let id: UUID
    var targetPlatforms: [DatingPlatform]
    var currentPhotos: [ProfilePhoto]
    var auditResult: ProfileAuditResult?
    var recommendedPhotoOrder: [UUID]
    var generatedPhotos: [GeneratedPhoto]
    var bio: String?
    var prompts: [PromptAnswer]
    var openers: [String]
    var exportChecklist: [ExportChecklistItem]
    var createdAt: Date
    var updatedAt: Date
}
```

Supporting models:

- `ProfilePhoto`
- `ProfileAuditResult`
- `MissingPhotoSlot`
- `PhotoRecommendation`
- `ExportChecklistItem`
- `PlatformProfileKit`

### Phase 3: Better Audit

Audit should evaluate the full photo set, not just individual images.

Dimensions:

- First-photo strength.
- Face clarity.
- Lighting.
- Expression.
- Crop.
- Variety.
- Authenticity.
- Platform fit.
- Missing profile slots.
- Potential AI/fake-looking risk.

Important:

Do not use generated-photo count as a proxy for profile quality.

### Phase 4: Generation UX

Replace AI model selection with user-visible outcomes:

- Natural.
- Polished.
- Candid.
- Dressed Up.
- Activity.
- Travel/Lifestyle.

Internally map these to providers, prompts, and models.

Advanced users can still access model controls from settings or an advanced sheet.

### Phase 5: Coach Upgrade

Add screenshot-based coach input for:

- Profile opener.
- Conversation reply.
- Revive dead chat.
- Ask for date.

This makes GigaRizz competitive with CupidAI and YourMove while preserving the photo-first wedge.

### Phase 6: Backend Contract Cleanup

Create one canonical API contract.

Required cleanup:

- Align docs, iOS client, and backend status names.
- Align routes for generation status.
- Ensure upload/presign contract is stable.
- Add staged provider mocks for deterministic tests.
- Add backend environment setup that works from a clean checkout.

### Phase 7: QA And Release Readiness

Required QA:

- First-run simulator UI test.
- Generate mock profile kit test.
- Real backend staging generation test.
- Upload/presign test.
- Paywall sandbox test.
- Restore purchases test.
- Delete data test.

- Accessibility pass for core flows.
- Small-screen layout pass.

## Current Issues To Address Before V2

### Onboarding Flag Mismatch

The docs describe server-driven onboarding, but the app currently routes fresh users into onboarding directly. V2 should make onboarding flags real and shorten the flow.

### Long First-Run Path

The 30-slide story flow delays the user from reaching the core value. Replace it with a diagnosis-first flow.

### Misleading Profile Score

The current profile score can be based on generated photo count and style count. V2 scoring must be based on actual photo and profile quality.

### Tool Sprawl

The current tools hub exposes too many separate capabilities. V2 should route users through profile outcomes and contextual actions.

### Progress Reporting

Generation progress should update while generation is happening, not after the service returns.

### Backend Test Reproducibility

Backend tests should run from a clean checkout with documented dependency setup.
